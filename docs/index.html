amesh GitHub

Stop managing API keys. Let the hardware prove identity.

amesh replaces static secrets with cryptographic device identity. The private key lives in your chip. There is nothing to leak.

What amesh replaces

Everything you store in .env files and pass around in Slack.

.env files Secrets on disk
API keys Static strings
Bearer tokens Shared secrets
JWT secrets Rotated manually

Before and after

Your code changes by one line. Your security model changes completely.

Today
# .env (leaked on GitHub)
API_KEY=sk_live_4eC39HqLyj...

fetch("/api/orders", {
  headers: {
    Authorization:
      `Bearer ${API_KEY}`
  }
});
With amesh
// No .env. No secrets anywhere.
import { amesh } from '@authmesh/sdk';

amesh.fetch("/api/orders", {
  method: "POST",
  body: JSON.stringify({
    amount: 100
  })
});

How it works

Four steps. Then every request is signed by hardware and verified cryptographically.

1

Create a device identity

Each machine gets a unique keypair. The private key never leaves the hardware.

$ amesh init --name "prod-api"

Identity created.
  Device ID : am_cOixWcOdI8-pLh4P
  Backend   : secure-enclave
2

Pair two machines

One runs amesh listen, the other amesh invite. A 6-digit code confirms no one is in the middle.

$ amesh listen

  Pairing code: 482916

✔ Controller connected.
  Verification code: 847291
  Codes match? (Y/n): y
✔ "Dev Laptop" added to allow list.
3

Sign requests — 2 lines

Import amesh and call amesh.fetch(). It reads your identity from ~/.amesh/ and signs automatically.

client.js 
import { amesh } from '@authmesh/sdk';

amesh.fetch("/api/orders", {
  method: "POST",
  body: JSON.stringify({ amount: 100 })
});
4

Verify requests — 2 lines

One line of middleware. It checks the signature, timestamp, nonce, and allow list.

server.js 
import { amesh } from '@authmesh/sdk';

app.use(amesh.verify());
// req.authMesh.deviceId available

Why this is better

Nothing to leak

No .env file. No secret in CI. No token in Slack. The key is in silicon.

Nothing to rotate

Device keys don't expire. Revoke a device instantly with amesh revoke.

Replay-proof

Every request has a unique nonce and a 30-second timestamp window.

Tamper-proof trust store

The allow list is HMAC-sealed. Any file edit is detected immediately.

MITM-proof pairing

Encrypted key exchange with 6-digit verification. Same as Signal and Bluetooth.

Manage devices from the terminal
